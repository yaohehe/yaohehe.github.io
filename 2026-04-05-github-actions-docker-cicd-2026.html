2026年GitHub Actions + Docker：CI/CD自动化部署完整实战

手动部署服务器的时代已经结束。2026年，如果你还在用FTP上传代码，或者每次发布都要登录服务器跑命令，你已经落后了。

本文手把手教你用GitHub Actions + Docker搭建完整的CI/CD流水线——代码提交到主分支，自动构建镜像，自动部署到VPS，生产环境分钟级生效。

为什么是GitHub Actions + Docker？

  • 零基础设施成本：GitHub Actions免费额度足够个人项目
  • 跨平台一致：本地build的镜像和服务器运行100%一致
  • 版本可控：每次部署有记录，回滚一键完成
  • 生态成熟：GitHub原生集成，配置YAML即可

前置条件

  • VPS（推荐DigitalOcean或Vultr，$4/月起）
  • GitHub账号（免费）
  • Docker已安装（1行命令）

第一步：在VPS上安装Docker

ssh root@你的VPS_IP
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker

第二步：创建GitHub Actions工作流

在你的项目根目录创建 .github/workflows/deploy.yml：

name: Build and Deploy

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to Docker Hub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_TOKEN }}

      - name: Build and Push
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: yourusername/yourapp:latest

      - name: Deploy to VPS
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.VPS_HOST }}
          username: root
          key: ${{ secrets.VPS_SSH_KEY }}
          script: |
            docker pull yourusername/yourapp:latest
            docker stop yourapp || true
            docker rm yourapp || true
            docker run -d --name yourapp -p 80:3000 yourusername/yourapp:latest

第三步：配置GitHub Secrets

在GitHub仓库 → Settings → Secrets and variables → Actions中添加：

  • DOCKER_USERNAME：Docker Hub用户名
  • DOCKER_TOKEN：Docker Hub Access Token
  • VPS_HOST：你的VPS IP地址
  • VPS_SSH_KEY：私钥（记得在VPS的~/.ssh/authorized_keys添加对应公钥）

第四步：写一个Dockerfile

FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build
EXPOSE 3000
CMD ["npm", "start"]

第五步：触发部署

只需一行命令：

git add .
git commit -m "feat: 自动化部署配置"
git push origin main

然后在GitHub仓库的Actions标签页，你会看到流水线实时运行——构建镜像→推送→SSH到VPS→拉取镜像→重启容器。全程无需人工干预，2-3分钟完成。

进阶：多环境部署

生产环境稳定后，可以扩展 staging 环境：

on:
  push:
    branches: [main]
  pull_request:
    branches: [develop]

常见问题

Q: Docker Hub免费额度够用吗？
A: 个人免费账户每月180次构建，对大多数项目足够。如需更多，可用GitHub Packages或自建Harbor。

Q: VPS配置低能跑Docker吗？
A: 建议至少1GB内存。Docker本身占用很少，主要看你的应用。

推荐VPS配置：DigitalOcean $4/月或Vultr $5/月，1GB内存+25GB SSD，首次部署完全够用。
