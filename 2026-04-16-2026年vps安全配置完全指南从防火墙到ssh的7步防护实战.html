VPS安全配置完全指南

vps安全服务器安全防火墙配置ssh加固vps教程

全球平均每39秒就有一次网络攻击。你的VPS，下一秒可能就是目标。

2025年的数据令人警醒：全球平均每39秒发生一次网络攻击，这意味着你的服务器在不为人知的角落正在遭受持续的入侵尝试。SSH暴力破解、恶意扫描、漏洞利用——如果你刚买了台VPS却不做任何安全配置，它很可能在24小时内就被自动化工具扫描并入侵，用于发送垃圾邮件、挖矿或发起DDoS攻击。

这不是危言耸听。这是每个VPS用户的真实处境。而最经济有效的解决方案，就是自己动手做安全配置。本文提供7个实战步骤，帮你把VPS安全风险降到最低。

第一步：修改SSH默认端口——最简单的第一道防线

SSH默认使用22端口，所有自动化攻击工具都知道这一点。把端口改到高位（如22022），能拦截80%以上的自动化暴力破解。

# 编辑SSH配置文件
sudo nano /etc/ssh/sshd_config

# 找到 Port 22 这一行，改为
Port 22022

# 重启SSH服务使配置生效
sudo systemctl restart sshd

：修改端口前，先开一个SSH会话保持连接，防止新会话连不上时失去控制权。新端口确认可用后，再关闭旧会话。

第二步：禁用密码登录，改用密钥认证

密码登录是暴力破解的主要入口。密钥认证不仅避免了弱密码问题，还能实现免密登录。

# 在本地生成SSH密钥对（如果你还没有）
ssh-keygen -t ed25519 -C "your_email@example.com"

# 把公钥复制到VPS
ssh-copy-id -p 22022 -i ~/.ssh/id_ed25519.pub user@your_vps_ip

# 编辑SSH配置，禁用密码登录
sudo nano /etc/ssh/sshd_config

# 确保以下配置：
PasswordAuthentication no
PubkeyAuthentication yes
PermitRootLogin no

ed25519密钥比传统的RSA密钥更安全、密钥更短，是2026年的推荐选择。生成后务必将私钥妥善保管——丢失私钥意味着失去服务器访问权限。

第三步：配置UFW防火墙——5分钟搭建基础防护

UFW（Uncomplicated Firewall）是Ubuntu默认的防火墙管理工具，命令行友好，适合快速配置。

# 安装并启用UFW
sudo apt update && sudo apt install ufw -y

# 设置默认规则：拒绝入站，允许出站
sudo ufw default deny incoming
sudo ufw default allow outgoing

# 允许SSH（使用你修改后的端口）
sudo ufw allow 22022/tcp

# 允许HTTP和HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 允许常见应用
sudo ufw allow 22/tcp    # 临时保留旧22端口，确认新端口可用后删除

# 启用防火墙
sudo ufw enable

# 查看状态
sudo ufw status verbose

对于Web服务器来说，开放80/443是最基本的。UFW的语法简洁直观，比直接操作iptables降低了出错概率。

第四步：安装并配置Fail2ban——自动封禁恶意IP

Fail2ban能监控SSH、HTTP等服务的日志，自动识别暴力破解行为并封禁对应IP。这是防御SSH暴力破解的自动化哨兵。

# 安装Fail2ban
sudo apt update && sudo apt install fail2ban -y

# 启动并设置开机自启
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

# 查看当前封禁状态
sudo fail2ban-client status
sudo fail2ban-client status sshd

Fail2ban的配置文件位于，可以根据需要调整封禁规则：

# 5次密码错误就封禁IP，封禁时间3600秒（1小时）
sudo nano /etc/fail2ban/jail.local

# 添加以下内容
[sshd]
enabled = true
port = 22022
maxretry = 5
findtime = 10
bantime = 3600

适当调高maxretry能减少误封，但不要设得太高，否则失去防护意义。

第五步：配置DDoS基础防护——Cloudflare是中小站长最优解

DDoS攻击可以轻易让没有防护的VPS下线。中小型网站最经济有效的方案是使用Cloudflare的免费CDN服务——它不仅能隐藏服务器真实IP，还能吸收DDoS攻击流量。

Cloudflare免费版的DDoS防护对中小流量网站已经足够。接入方式也很简单：

1. 在Cloudflare注册账号并添加你的域名

2. 将域名的NS服务器修改为Cloudflare提供的地址

3. 在Cloudflare控制台中开启"Under Attack Mode"（遭遇攻击模式）

接入Cloudflare后，你的服务器IP被隐藏，攻击者只能打到Cloudflare的边缘节点，你的源站IP不会被暴露。配合UFW限制源IP访问，能进一步减少攻击面。

对于国内业务，腾讯云也提供DDoS基础防护服务，适合已经使用腾讯云服务的企业用户。

第六步：开启自动安全更新——别让漏洞敞开超过24小时

Linux发行版会定期发布安全补丁，但不会自动安装。配置 unattended-upgrades，让系统自动下载并安装安全更新。

# 安装自动更新包
sudo apt install unattended-upgrades update-notifier snaps -y

# 配置自动更新
sudo dpkg-reconfigure -plow unattended-upgrades

# 手动编辑配置文件
sudo nano /etc/apt/apt.conf.d/50unattended-upgrades

# 启用自动安全更新
Unattended-Upgrade::Allowed-Origins {
    "${distro_id}:${distro_codename}-security";
};

配置后，系统会在后台自动安装安全更新，你可以在查看更新日志。建议每月检查一次日志，确保没有异常。

第七步：定期备份——最后一道防线

即便做了所有防护，也无法保证100%安全。定期备份才是真正的最后一道防线。

# 使用rsync备份到远程服务器
rsync -avz -e "ssh -p 22022" /var/www/html/ user@backup-server:/backup/

# 或者使用rclone备份到云存储（以腾讯云COS为例）
rclone copy /var/www/html/ cos://your-bucket/backup/ --progress

建议策略：

  • **每日增量备份**：自动执行，保留最近7天
  • **每周全量备份**：上传到独立云存储
  • **重要操作前手动备份**：修改配置、安装软件前先做快照

腾讯云的快照功能非常方便，按量计费，成本可控。Hostwinds用户可享受每周免费快照，值得利用。

总结：安全是一场持续的战斗

VPS安全不是"设一次就完事"的静态配置，而是需要持续运营的动态过程：

  • **端口修改+密钥认证**是最基础的SSH防护，5分钟就能完成
  • **UFW防火墙**是网络层的第一道门，选择性开放端口
  • **Fail2ban**是自动化安保，帮你实时监控并封禁攻击者
  • **Cloudflare/腾讯云**的DDoS防护是应对大规模攻击的必备
  • **自动更新+定期备份**是最后的安全垫，再完善的防护也可能被攻破

以上7步不需要额外付费买软件，系统自带工具就能构建相对完善的安全体系。安全配置的边际成本极低——你投入的1-2小时，能换来服务器长期的稳定运行和数据安全。
