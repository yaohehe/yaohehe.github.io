# 内容质量评分标准

每次生成文章后，对照以下5项评分，每项1-5分，总分≥18分才能发布。

## 1. 标题吸引力（5分）
- 5分：数字明确+情感触发+关键词前置（如"2026年最佳VPS：8年实测数据告诉你哪款最值得"）
- 4分：有数字+关键词，但情感触发一般
- 3分：有关键词，格式规范但平淡
- 2分：标题含糊，无具体信息
- 1分：标题与内容不符或关键词堆砌

## 2. 文章结构（5分）
- 5分：H2层次清晰（≥4个章节），每个章节有小结，段落≤3行
- 4分：H2≥3个，段落≤4行
- 3分：有章节标题但层次混乱
- 2分：通篇无小标题或只有2个
- 1分：流水账式写作，无结构

## 3. 内容深度（5分）
- 5分：包含实测数据/案例/具体价格/对比表格，有第三方来源
- 4分：内容全面但缺少数据支撑
- 3分：有观点但深度不足（通篇都是泛泛而谈）
- 2分：大量套话，无实际可操作内容
- 1分：与其他文章高度重复，无差异化

## 4. 读者参与度（5分）
- 5分：开头有痛点场景引发共鸣，结尾有明确CTA，联盟链接植入自然不像广告
- 4分：有场景开头，CTA清晰，联盟链接较自然
- 3分：有开头但结尾弱，联盟链接较明显
- 2分：开头沉闷，结尾无CTA，全文像软文
- 1分：纯粹产品推销，无可读价值

## 5. SEO 规范性（5分）
- 5分：关键词自然分布，元数据完整（描述/标签/一级标题），内链有逻辑
- 4分：关键词分布合理，元数据完整
- 3分：关键词有堆砌感或分布不均，元数据部分缺失
- 2分：只强插关键词，元数据随意
- 1分：无元数据或完全关键词堆砌

---

## 发布阈值
- 总分≥18分 → ✅ 可发布
- 总分15-17分 → ⚠️ 需修改后发布
- 总分<15分 → ❌ 重写

## 生成技巧备忘（从实践中提炼）

### 文章类型适配

**对比类文章：**
- 开头直接放对比结论（省时间者必看）
- 必须有对比表格（价格/配置/优缺点）
- 结尾给出"适合人群"建议

**推荐类文章：**
- 包含价格区间+适用人群+缺点说明
- 禁止只说"好"，要说明"哪里好"和"可能的坑"

**教程类文章：**
- 步骤≤10步，每步有具体命令或操作
- 包含常见错误和解决方案
- 适合人群说明

**联盟文章：**
- 先提供真实价值（帮助读者解决问题）
- 再自然引出推荐（不是硬推销）
- 声明联盟关系（透明原则）

## 内容质量检查清单（content-writer 融合版）

生成后必须逐项检查：

- [ ] **标题**：是否足够吸引人？有数字和关键词？
- [ ] **开头hook**：前3行是否能抓住读者？是否有痛点场景？
- [ ] **内容深度**：是否有具体例子/数据/案例？不是泛泛而谈？
- [ ] **结构清晰**：是否有≥3个小标题？段落是否≤4行？
- [ ] **有CTA**：结尾是否有明确行动指引？
- [ ] **E-E-A-T达标**：
  - [ ] Experience（经验）：是否写了"我实测了X个月"？
  - [ ] Expertise（专业）：是否有技术细节/命令/数据？
  - [ ] Authoritativeness（权威）：是否引用了第三方来源？
  - [ ] Trustworthiness（可信）：是否说明了不适用人群？
- [ ] **字数达标**：中文≥1500字，英文≥800词
- [ ] **SEO关键词**：是否自然分布？不是堆砌？
- [ ] **违禁词检查**：无绝对化断言（最好/第一/唯一）
- [ ] **链接真实**：Amazon链接ASIN是否验证过？

---

## 内容来源与验证原则

**✅ 可以写的：**
- 搜索验证过的真实产品信息
- 公开可查的数据（官方文档、权威评测）
- 个人经验（注明"我用了X个月"、"我测试了..."）

**❌ 禁止写的：**
- 没搜索过的ASIN或产品
- 夸大其词的绝对化表述（"最好的"、"第一名的"）
- 伪造的个人经历
- 无法验证的价格或数据

**⚠️ 不确定时的处理：**
- 价格：注明"截至发稿前价格，请以实际为准"
- 产品特性：注明"建议自行确认"
- 统计数据：注明来源（如"根据 Statista 2025 报告"）
