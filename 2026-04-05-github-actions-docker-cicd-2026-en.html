GitHub Actions + Docker CI/CD in 2026: Complete Automation Guide

Manual server deployments are dead. In 2026, if you're still using FTP to upload code or logging into servers to run deployment commands, you're behind.

This guide walks you through setting up a complete CI/CD pipeline with GitHub Actions + Docker—push to main branch, Docker image builds automatically, deploys to your VPS, production goes live in minutes.

Why GitHub Actions + Docker?

  • Zero infrastructure cost: GitHub Actions free tier covers personal projects
  • Consistent across environments: What builds locally runs identically in production
  • Version controlled: Every deployment logged, one-click rollback
  • Mature ecosystem: Native GitHub integration, YAML-only configuration

Prerequisites

  • A VPS (recommend DigitalOcean or Vultr, from $4/mo)
  • GitHub account (free)
  • Docker installed on your VPS (one command)

Step 1: Install Docker on Your VPS

ssh root@your_vps_ip
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker

Step 2: Create the GitHub Actions Workflow

Create .github/workflows/deploy.yml in your project root:

name: Build and Deploy

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to Docker Hub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_TOKEN }}

      - name: Build and Push
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: yourusername/yourapp:latest

      - name: Deploy to VPS
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.VPS_HOST }}
          username: root
          key: ${{ secrets.VPS_SSH_KEY }}
          script: |
            docker pull yourusername/yourapp:latest
            docker stop yourapp || true
            docker rm yourapp || true
            docker run -d --name yourapp -p 80:3000 yourusername/yourapp:latest

Step 3: Configure GitHub Secrets

Go to GitHub repo → Settings → Secrets and variables → Actions, add:

  • DOCKER_USERNAME: Docker Hub username
  • DOCKER_TOKEN: Docker Hub Access Token
  • VPS_HOST: Your VPS IP address
  • VPS_SSH_KEY: Private key (add corresponding public key to VPS ~/.ssh/authorized_keys)

Step 4: Write a Dockerfile

FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build
EXPOSE 3000
CMD ["npm", "start"]

Step 5: Trigger Your First Deployment

One command:

git add .
git commit -m "feat: add CI/CD automation"
git push origin main

Head to the Actions tab in your GitHub repo—you'll see the pipeline run live: build image → push → SSH to VPS → pull → restart container. Done in 2-3 minutes, zero manual intervention.

Pro: Multi-Environment Deployments

After stabilizing production, extend to staging:

on:
  push:
    branches: [main]
  pull_request:
    branches: [develop]

FAQ

Q: Is Docker Hub free tier enough?
A: Personal free tier gives 180 builds/month—plenty for most projects. For more, use GitHub Packages or self-host Harbor.

Q: Can low-spec VPS run Docker?
A: 1GB RAM minimum recommended. Docker itself is lightweight—your app is the bigger factor.
