## 什么是 OpenClaw？

OpenClaw 是一款基于大模型的 AI Agent 运行时环境，支持定时任务、会话管理、插件扩展和跨平台集成。与传统的聊天式 AI 不同，OpenClaw 被设计为无人值守的自动化引擎——你设定规则，AI 执行一切。

核心特性：

  • Cron 定时调度：支持标准 cron 表达式，可精确控制执行时间
  • Isolated Session：每个任务在独立上下文中运行，避免历史干扰
  • 插件生态：支持企业微信、QQ、Telegram 等多平台消息通道
  • 失败自动重试：配合 failureAlert 实现完整的告警-恢复闭环

对于 TechPassive 这样的技术内容博客，OpenClaw 扮演的是「数字员工」的角色——它不知疲倦、不情绪化、按时按质完成任务。

## 为什么 TechPassive 需要自动化？

手动运营一个技术博客的问题是显而易见的：

时间成本高：每次写一篇 SEO 友好的中英双语文章，加上研究、配图、HTML 转换、发布的全流程，保守估计需要 3-4 小时。按照每周发布 2 篇的频率，每月投入超过 24 小时在重复性工作中。

一致性难维持：人脑会疲劳，状态会有波动。手动发布容易出现更新断档、格式不统一、SEO 参数遗漏等问题。

规模化受限：想从每周 2 篇扩展到每天 2 篇？单纯靠人工几乎不可能。

自动化解决了这三个核心痛点。你只需要做三件事：设定规则、验证结果、处理异常。一切重复性工作交给 AI。

## 搭建自动化流水线的核心技术栈

### 1. 定时任务调度（OpenClaw Cron）

`bash

每天 14:00 UTC 执行内容生成

0 14 * → generate-article

# 每天 22:00 UTC 执行发布 0 22 * → publish-article `

OpenClaw 的 cron 支持精确到分钟的任务调度。建议将内容生成和发布拆分为独立任务——这样做的好处是单点故障不会传染，生成失败不会导致发布环节受影响。

### 2. 内容生成（Python + AI）

内容生成的核心逻辑其实很简单：

`python

generate-html.py 核心流程

1. 读取 /tmp/article-gen/cn.txt 和 en.txt 2. 注入百度统计脚本（前） 3. 注入 AdSense 代码（
后 + 前） 4. 输出到 drafts/ 目录 `

HTML 模板由 Python 脚本统一处理，确保每篇文章的统计和广告注入规则完全一致。这是消除人为错误的关键——让机器做格式化的事情。

### 3. 主题轮换机制

为了保证内容多样性，同时避免重复，我们建立了 5 大主题池：

| 序号 | 主题 | 覆盖领域 | |------|------|----------| | 1 | WordPress 建站教程 | 域名、主机、主题、插件 | | 2 | DevOps 自动化 | CI/CD、容器化、自动化部署 | | 3 | 联盟营销实战 | 选品、推广、佣金追踪 | | 4 | AI 和 OpenClaw 自动化盈利 | Agent 工具、变现路径 | | 5 | 云服务器/VPS/云计算 | AWS/GCP/Azure/Vultr/DO 对比 |

每次生成任务从池中挑选上次未使用的主题，通过简单的数组索引偏移实现轮换。关键代码只需要几行：

`python last_topic = read_last_topic_from_memory() next_index = (topics.index(last_topic) + 1) % len(topics) `

### 4. 失败告警与自动恢复

这是自动化系统的灵魂。再精密的系统也会出错，关键在于错误能否被快速发现并自动修复。

`yaml failureAlert: after: 2 # 连续2次失败才告警 channel: lightclawbot to: "1407429596" `

配合自动重跑机制，发布失败后系统会自动重新尝试，2次全部失败才会通知你。这意味着偶尔的网络抖动不会打扰你，但真正的故障一定会被看到。

## 实战：完整流水线的 24 小时

让我们过一遍这条流水线的一天：

14:00 UTC（22:00 北京时间）：定时任务触发，OpenClaw 在 isolated session 中启动 AI 生成任务。AI 根据当天应选主题生成中文正文（≥1500字）和英文正文（≥800词），写入临时文件。生成过程无需人工干预，AI 自动注入 SEO 元数据、百度统计和 AdSense 代码。

14:05 UTC：Python 脚本 generate-html.py 执行，将纯文本转换为标准 HTML 格式。输出文件按命名规则存放：YYYY-MM-DD-主题-slug.html 和 YYYY-MM-DD-主题-slug-en.html。

22:00 UTC（次日 06:00 北京时间）：发布任务启动，Python 脚本将草稿推送到 GitHub Pages。推送完成后，系统自动更新 sitemap.xml 和博客首页索引。整个过程耗时不超过 30 秒。

次日 14:00 UTC：新一轮循环启动。如果昨天的发布成功，系统正常执行；如果失败，failureAlert 机制在连续2次失败后通知你，你登录检查，修复问题，下次执行自动恢复正常。

## 关键配置清单

如果你要复制这套系统，以下是你需要配置的清单：

1. OpenClaw 环境：安装 openclaw-cli，配置 gateway 和 channel 插件 2. 定时任务：通过 openclaw cron add 创建生成任务和发布任务 3. failureAlert：为每个 cron 任务配置连续失败告警 4. GitHub 仓库：初始化空的 gh-pages 分支，配置 Deploy Key 5. 百度统计：注册百度统计账号，获取 hm.js 跟踪代码 6. AdSense：申请 AdSense，配置 pub-xxx 发布商 ID 和广告位 7. TencentDB Memory（可选）：跨 session 持久化主题轮换状态

## 收益与局限

这套自动化流水线的价值在于释放你的时间，让你专注在高杠杆的事情上——比如研究新的变现路径、分析流量数据、设计内容策略。

但它不是银弹。内容的质量最终取决于你设定的规则和 AI 的生成能力。自动化可以保证一致性和频率，但无法替代你对内容价值的判断。

下一步建议：先从手动触发一次完整流程开始，验证每个环节的输出，再切换到定时自动化模式。纸上谈兵不如实战验证。
