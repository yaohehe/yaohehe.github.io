## 引言：你是否还在用FTP手动部署WordPress？

如果你有一个自建的WordPress网站，每次更新主题、修改插件或者部署新文章时，还要手动用FTP上传文件——那这篇文章就是为你写的。作为一名技术博主，我花了很长时间才从"改完文件→FTP上传→检查是否生效"的痛苦循环中走出来。今天，我要分享如何用 GitHub Actions 实现 WordPress的CI/CD自动化部署。

## 为什么WordPress也需要CI/CD？

很多人觉得WordPress只是一个"博客平台"，不需要CI/CD。但实际上：

- 主题和插件版本管理混乱：直接在生产环境修改代码，出了问题无法回滚

  • 手动部署容易出错：上传遗漏文件、覆盖数据库、缓存未清理
  • 团队协作困难：多个人同时修改主题文件，没有版本控制
  • 部署不可追踪：不知道谁在什么时候改了什么，出了问题无从排查

引入CI/CD后，你的WordPress部署流程会变成这样：本地修改主题代码 → 推送到GitHub → 自动测试 → 自动部署到服务器 → 自动清理缓存。全程无需手动干预，零停机、可回滚、可追溯。

## 第一步：将WordPress主题版本化

首先，你需要把WordPress主题（或子主题）纳入Git版本控制。这里推荐使用子主题（Child Theme），这样父主题更新不会覆盖你的修改。

`bash cd /path/to/wordpress/wp-content/themes

如果你还没有子主题，先创建一个

mkdir -p mytheme-child cd mytheme-child `

创建子主题的三个必需文件：

style.css — 子主题的声明文件： `css /* Theme Name: MyTheme Child Template: twentytwentyfour Version: 1.0.0 */ `

functions.php — 用于引入父主题样式和自定义代码： `php `

screenshot.png — 主题预览图（可选，放在WordPress后台主题列表中显示）

初始化Git仓库： `bash git init git remote add origin git@github.com:yourusername/wp-theme.git git add . git commit -m "初始化子主题" git push -u origin main `

## 第二步：配置服务器上的免密部署

GitHub Actions需要通过SSH连接到你的服务器来部署文件。配置步骤如下：

在服务器上生成SSH密钥对（如果还没有的话）： `bash ssh-keygen -t rsa -b 4096 -f ~/.ssh/deploy_rsa -N "" `

将公钥添加到服务器的authorized_keys： `bash cat ~/.ssh/deploy_rsa.pub >> ~/.ssh/authorized_keys chmod 600 ~/.ssh/authorized_keys `

将私钥添加到GitHub Secrets： 1. 进入你的GitHub仓库 → Settings → Secrets and variables → Actions 2. 点击 "New repository secret" 3. Name填 SSH_PRIVATE_KEY 4. Value填入私钥内容：cat ~/.ssh/deploy_rsa

同时建议添加以下Secrets：

  • SERVER_HOST：你的服务器IP或域名
  • SERVER_USER：SSH用户名（如deploy）

## 第三步：编写GitHub Actions工作流

在你的主题仓库中创建 .github/workflows/deploy.yml：

`yaml name: Deploy WordPress Theme

on: push: branches: [main]

# 也支持手动触发 workflow_dispatch:

jobs: deploy: runs-on: ubuntu-latest steps: - name: 检出代码 uses: actions/checkout@v4

- name: 部署到WordPress服务器 uses: appleboy/ssh-action@v1.0.3 with: host: ${{ secrets.SERVER_HOST }} username: ${{ secrets.SERVER_USER }} key: ${{ secrets.SSH_PRIVATE_KEY }} script: | THEME_DIR="/var/www/html/wp-content/themes/mytheme-child" DEPLOY_DIR="/tmp/wp-deploy-$$"

# 创建部署目录 mkdir -p $DEPLOY_DIR

# 将代码推送到部署目录 cd $DEPLOY_DIR # 这里用rsync从GitHub拉取会比较复杂 # 更推荐的方式是用rsync直接push

- name: Rsync部署 uses: burnett01/rsync-deployments@v7 with: switches: -avzr --delete path: "./" remote_path: "/var/www/html/wp-content/themes/mytheme-child" remote_host: ${{ secrets.SERVER_HOST }} remote_user: ${{ secrets.SERVER_USER }} remote_key: ${{ secrets.SSH_PRIVATE_KEY }}

- name: 清理缓存 uses: appleboy/ssh-action@v1.0.3 with: host: ${{ secrets.SERVER_HOST }} username: ${{ secrets.SERVER_USER }} key: ${{ secrets.SSH_PRIVATE_KEY }} script: | # 如果使用WP-CLI cd /var/www/html wp cache flush --allow-root # 如果用了Redis/Memcached # redis-cli FLUSHDB echo "✅ 部署完成，缓存已清理" `

## 第四步：优化你的部署流程

上面的基础工作流已经可以用了，但实际运营中建议添加以下优化：

1. 添加部署通知

`yaml - name: 发送部署通知 if: always() run: | curl -X POST "https://your-webhook-url/notify" \ -H "Content-Type: application/json" \ -d "{\"status\":\"${{ job.status }}\",\"commit\":\"${{ github.sha }}\"}" `

2. 部署前运行代码检查

`yaml - name: PHP代码检查 run: | docker run --rm -v $(pwd):/app ghcr.io/phpstan/phpstan analyse . `

3. 实现回滚机制

`yaml - name: 创建部署前备份 uses: burnett01/rsync-deployments@v7 with: switches: -avz remote_path: "/var/www/html/wp-content/themes/mytheme-child/" path: "/tmp/wp-backup/$(date +%Y%m%d-%H%M%S)/" `

## 常见问题与解决方案

Q: rsync部署失败，提示permission denied？

确保你的SSH用户有写入WordPress主题目录的权限。可以： `bash sudo chown -R deploy:www-data /var/www/html/wp-content/themes/mytheme-child sudo chmod -R 755 /var/www/html/wp-content/themes/mytheme-child `

Q: 更新后前台没有变化？

很可能是缓存问题。除了上面的 wp cache flush，还要检查：

  • 对象缓存（Redis/Memcached）
  • 页面缓存插件（WP Super Cache/W3 Total Cache）
  • CDN缓存（Cloudflare等）

Q: 如何管理数据库变更？

主题文件可以用Git管理，但数据库变更需要额外工具。推荐：

  • WP Migrate DB Pro：同步数据库和本地文件
  • 自定义WP-CLI脚本：在部署后执行数据库迁移
  • WordPress REST API：通过API管理内容变更

## 总结

用GitHub Actions实现WordPress自动化部署，核心价值在于：

- 可追溯：每一次部署都有记录，随时回滚

  • 零停机：rsync增量同步，不影响现有访问
  • 可扩展：后续轻松添加代码检查、自动备份、缓存清理
  • 省时省力：改完代码push即可，不用手动FTP

作为TechPassive博客的站长，我认为自动化运维是每一个WordPress站长的必修课。投入几个小时搭建好这套流程，后续每天节省的时间会让你觉得非常值得。
